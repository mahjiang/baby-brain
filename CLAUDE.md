# CLAUDE.md — BabyBrain Project

## Project Purpose

This is Eric's baby prep research and planning hub. Eric is a first-time parent; his partner is currently ~21 weeks pregnant (mid-second trimester, due ~July 2026). They live in a US urban/suburban apartment with 2 cats, use both car and transit, and have no dedicated nursery room. Both parents get paid leave. Budget is moderate.

## Eric's Profile

- Primary researcher — does deep dives, then discusses decisions with partner
- Evidence-based planner who is thorough but currently feeling overwhelmed
- Prefers new for safety-critical items (car seat, crib), used is fine for clothes/toys
- Baby shower is expected, so registry strategy matters
- There are some pregnancy complications being monitored
- Strongly prefers buy-once solutions — minimize waste and reselling, but not at the cost of quality for convertibility

## File Conventions

### Research Format
When adding research to any file:
- Use `## Research` or `## Evidence` sections
- Cite sources inline: `(Source: [AAP Safe Sleep Guidelines](url))`
- Prefer peer-reviewed sources, AAP, WHO, and trusted evidence-based authors (Emily Oster, etc.)
- Flag anecdotal information explicitly: `*Anecdotal:*`
- Flag opinion vs evidence: `*Opinion:*` vs `*Evidence:*`
- Date-stamp new research additions: `*Added YYYY-MM-DD*`

### Decision Format
Major purchases and choices use this format (see `09-decisions/_template.md`):
- **Status**: `Researching` | `Ready to discuss` | `Decided`
- Include options with pros/cons
- Include evidence/sources
- Include "Our situation" section (apartment, cat, transit, budget factors)
- Eric fills in recommendation; final decision is joint

### Updates
- **Keep files current** — Files should reflect the best current knowledge and decisions. Replace outdated info rather than preserving it.
- Git history is the decision trail — no need for strikethrough or inline change logs
- Checkboxes `- [ ]` for action items, `- [x]` when done

### File Structure
- Each file starts with `# Title`
- Use `## Sections` for major divisions
- Use `> Blockquotes` for key takeaways or partner discussion prompts
- Use `---` to separate major sections
- Cross-link to related files with relative paths: `[essentials](02-gear/essentials.md)`

## Tone

- Direct, concise, no fluff
- Scannable — use bullet points, tables, and headers liberally
- Flag uncertainty honestly ("evidence is mixed," "no strong data")
- No judgment about parenting choices — present evidence and let them decide
- Practical over aspirational — what actually helps, not what looks good on Instagram

## Key Files

- `timeline.md` — The single source of truth for "what to do this week"
- `02-gear/essentials.md` — Master buy list with status tracking
- `02-gear/registry-strategy.md` — Registry approach
- `09-decisions/_template.md` — Template for decision docs

## How to Help

When Eric asks about a topic:
1. Check if a relevant file already exists in the project
2. Use the `/baby-research` skill for any research task — it provides curated sources, situation filtering, and the Oster evidence framework
3. Add research to the existing file (don't create new files unless needed)
4. Use the decision doc format for purchase/choice decisions
5. Update `timeline.md` if timing-relevant tasks are identified
6. Cross-link between files when relevant
